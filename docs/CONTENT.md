# Работа с контентом: CMS, статьи, страницы

Весь контент сайта — обычные файлы в git: Markdown в `src/content/` и один YAML с резюме. Ни базы данных, ни внешней CMS нет. Редактировать можно двумя способами:

1. **Keystatic** — визуальная админка с формами и WYSIWYG-редактором (рекомендуется);
2. руками в любом редакторе — это те же файлы.

Оба способа заканчиваются одинаково: **commit → push в `main` → кнопка деплоя** (GitHub → Actions → «Deploy (build on VPS)» → Run workflow, ~2–3 минуты). Push сам по себе на сайт ничего не выкатывает — можно накопить несколько правок и выкатить разом. Пока вы не сделали push, изменения существуют только у вас локально.

## Запуск админки

```bash
npm run dev
```

- Сайт-превью: http://localhost:4321 (обновляется на лету при любой правке)
- Админка: **http://localhost:4321/keystatic**

Админка работает только в dev-режиме на вашей машине. На боевом сайте её нет — это сознательное решение: нечего взламывать, нечему падать.

## Разделы админки

| Раздел | Файл(ы) | Что это |
| --- | --- | --- |
| Статьи блога | `src/content/blog/*.md` | посты |
| Страница «Обо мне» | `src/content/pages/about.md` | /about |
| Страница «Now» | `src/content/pages/now.md` | /now + тизер на главной |
| Резюме | `src/content/resume.yaml` | /resume |

(Английская версия «Обо мне» — `src/content/pages/en/about.md` — правится руками, она меняется редко.)

## Статьи

### Создать через админку

«Статьи блога» → **Create**. Поля:

- **Заголовок** — человекочитаемый; рядом автоматически генерится slug (имя файла и кусок URL — можно поправить руками, лучше латиницей);
- **Описание** — одно предложение до ~155 символов. Показывается в карточке, в Google-сниппете и соцсетях — пишите как мини-анонс, а не «в этой статье рассказывается…»;
- **Дата публикации** — по ней сортируется лента;
- **Дата обновления** — заполняйте только при существенной правке уже опубликованного;
- **Теги** — 1–3 штуки, в нижнем регистре; смотрите существующие на /blog, не плодите синонимы («команды» и «команда» станут разными тегами);
- **Черновик** — пока включён, статья видна в dev-превью, но **не попадает** на боевой сайт, в RSS и sitemap. Можно спокойно пушить недописанное;
- **Язык** — ru/en, влияет на форматирование дат и `lang` разметки;
- **Текст статьи** — WYSIWYG: заголовки H2/H3, списки, цитаты, код-блоки, таблицы, ссылки. Если заголовков H2/H3 три и больше — на странице статьи автоматически появится оглавление.

### Создать из терминала

```bash
npm run new:post -- "Заголовок новой статьи"
```

Создаст `src/content/blog/zagolovok-novoy-stati.md` (транслит) с готовым frontmatter, сегодняшней датой и `draft: true`.

### Правила frontmatter (при ручном редактировании)

```yaml
---
title: 'Заголовок: если внутри двоеточие — обязательно кавычки'
description: Одно предложение до 155 символов.
pubDate: 2026-07-08
updatedDate: 2026-07-10   # опционально
tags: [менеджмент, iot]
draft: false
lang: ru
---
```

Ошибка в frontmatter (опечатка в имени поля, неверная дата, двоеточие без кавычек) **валит сборку** с сообщением, какое поле и в каком файле не так. Это защита: сломанный контент не доедет до production. Схема-источник истины: [src/content.config.ts](../src/content.config.ts).

### Картинки в статьях

- Простой способ: положите файл в `public/images/` и вставьте `![альт-текст](/images/file.jpg)`.
- С оптимизацией (webp/avif, ленивая загрузка): положите рядом со статьёй, например `src/content/blog/images/schema.png`, и ссылайтесь относительно: `![альт-текст](./images/schema.png)`.

Всегда пишите осмысленный альт-текст.

### Публикация

```bash
git add -A
git commit -m "Статья: как мы чинили телеметрию"
git push
```

Затем GitHub → **Actions** → «Deploy (build on VPS)» → **Run workflow** — через 2–3 минуты статья на https://petriyov.io.

## Страница Now — самая частая операция

Раз в 3–6 недель: админка → «Страница Now». Обновите **три вещи**:

1. текст (что изменилось);
2. **«Тизер для главной»** — одна строка, она видна на главной под био («Сейчас: …»);
3. **«Дата обновления»** — админка подсвечивает это поле подсказкой; несвежая дата на now-странице хуже её отсутствия.

Сохранить → commit → push → кнопка деплоя в Actions.

## Обо мне

Админка → «Страница „Обо мне"». Не забывайте: обе формы фамилии (Петриёв / Petriyov) и ключевые роли должны естественно встречаться в тексте — это часть SEO-стратегии. Английскую версию `src/content/pages/en/about.md` при больших правках стоит синхронизировать по смыслу (не обязательно дословно).

## Резюме

Админка → «Резюме»: роль, краткое описание, опыт (карточки можно перетаскивать), навыки по группам, образование. Или руками в [src/content/resume.yaml](../src/content/resume.yaml) — помните про кавычки вокруг строк с двоеточием.

PDF-версия — отдельный файл `public/petriyov-cv.pdf`: замените его вручную, из данных сайта он не генерируется.

## Контакты, меню, имя, соцссылки

Это не контент, а конфиг: [src/config/site.ts](../src/config/site.ts). Меняются в одном месте, подхватываются шапкой, футером, страницей контактов, SEO-разметкой и JSON-LD.

## Проверка перед публикацией

- Быстро: `npm run dev` и глазами.
- Строго (ровно то, что увидит посетитель): `npm run build`, а лучше полный прогон в Docker — см. [docs/DOCKER.md](DOCKER.md).
